# Lodge & Grimoire Redesign

## Summary

Redesign the Lodge from a monitoring dashboard into an active workspace / bulletin board. Quests and humans post typed cards (notes, checklists, meetings, alerts, links, proposals, augury). The Grimoire becomes a quest log with per-quest run history and scoped telemetry. Absorb ex_cellence_ui components into the main app.

## Lodge: Active Workspace

### Card Schema (`lodge_cards`)

| Column     | Type     | Description |
|------------|----------|-------------|
| type       | string   | `"note"`, `"checklist"`, `"meeting"`, `"alert"`, `"link"`, `"proposal"`, `"augury"` |
| title      | string   | Card title |
| body       | string   | Markdown content (rendered via MDEx) |
| metadata   | jsonb    | Type-specific data (checklist items, meeting attendees, link URL, proposal details) |
| pinned     | boolean  | Pinned cards float to top |
| source     | string   | `"manual"`, `"quest"`, or quest name |
| quest_id   | integer  | Optional FK to quest that posted it |
| status     | string   | `"active"`, `"dismissed"`, `"archived"` |
| timestamps |          |  |

### Card Types

- **Note** — freeform markdown content (pinned findings, summaries, announcements)
- **Checklist** — titled list with checkable items in `metadata.items[].{text, checked}`
- **Meeting** — agenda + attendees + notes in metadata
- **Alert** — time-sensitive notification, visually distinct
- **Link** — bookmarked URL with title/description
- **Proposal** — learning loop suggestions with approve/reject actions (migrated from current Lodge)
- **Augury** — guild's "living read on the world" (migrated from Grimoire), pinned at top

### Card Creation Paths

1. **Quests** call `Lodge.post_card/1` from step runners
2. **Users** create via "+ Add" button on Lodge UI
3. **Proposals** auto-created by the learning loop (same as today, now as cards)
4. **Augury** single pinned card, editable in-place

### Card Interactivity

- Checklists: toggle items via `phx-click` updating `metadata.items[n].checked`
- Proposals: approve/reject buttons
- Augury: edit-in-place markdown
- Notes/alerts: pin/dismiss/archive actions
- All cards: delete action

### Lodge as Source Type

A `"lodge"` source type in the Sources system. Quests can subscribe to Lodge cards as input, filtered by type, tags, status. Example: a quest scans all pinned notes or open checklist items for context.

## Grimoire: Quest Log & Replay

### Two-Level Navigation

1. **Overview tab** — global aggregation across all quests. Houses the relocated monitoring widgets:
   - ReplayViewer (recent decisions)
   - OutcomeTracker (resolution stats)
   - DriftMonitor (action distribution shifts)
   - CalibrationChart (confidence calibration)
   - AgentHealth (member/model status)
   - Member Trust scores

2. **Per-quest view** — click a quest to see:
   - Run history (step outputs per run)
   - Lore entries generated by this quest
   - Decisions made during this quest's runs
   - Outcomes scoped to this quest
   - Same monitoring widgets, filtered to this quest's data

### Lore Entries Stay Here

Lore entries are knowledge artifacts from quest runs — they belong in the historical record. The Drop In intake form stays on the Grimoire (feeds content into quests).

### No New Schema Needed

Existing `lore_entries`, `decisions`, `outcomes`, `step_runs` tables already have quest/step FKs for scoping. This is primarily a UI reorganization.

## ex_cellence_ui Absorption

### Move Components

Absorb the 6 ex_cellence_ui components into `lib/ex_cortex_web/components/excellence/`:
- RoleForm
- ActionsForm
- GuardForm
- PipelineBuilder
- AIBuilder
- CharterPicker

Delete the standalone `ex_cellence_ui/` project (its own `.git`, `_build`, `deps`, `mix.lock`). Remove path dep from `mix.exs`.

### New Card Renderer

`ExCortexWeb.Components.LodgeCards` — single module with `lodge_card/1` function component that pattern-matches on card type:

```elixir
def lodge_card(%{card: %{type: "note"}} = assigns), do: ~H"..."
def lodge_card(%{card: %{type: "checklist"}} = assigns), do: ~H"..."
def lodge_card(%{card: %{type: "meeting"}} = assigns), do: ~H"..."
```

Adding a new card type = new function clause + metadata shape. No schema migration.

## Migration Path

1. Absorb ex_cellence_ui into main app
2. Create `lodge_cards` table + context
3. Build card renderers (one per type)
4. Rebuild Lodge LiveView as card workspace
5. Migrate proposals to card type
6. Move Augury from Grimoire to Lodge as pinned card
7. Add Lodge source type for quest consumption
8. Add manual card creation UI
9. Rebuild Grimoire as quest log with overview + per-quest tabs
10. Move monitoring widgets from Lodge to Grimoire
11. Scope monitoring widgets per-quest in drill-down view
